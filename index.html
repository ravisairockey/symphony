Using Particle Symphony

Browser Requirements

Use a WebGL2 capable browser. Chrome or Edge with a discrete GPU is recommended for 524k to 1M particles. Integrated GPUs are auto-protected by the particle budget selector and low-FPS reducer.

Open And Deploy

This is a single index.html. Double-click it for local use, or deploy the file directly to GitHub Pages, Netlify, Vercel, or any static host. Microphone access may require HTTPS or localhost.

Audio Sources

Click Start microphone for live input, click Load track for an audio file, or drag and drop a file on the viewport. Browser autoplay rules require one user gesture before audio starts.

Shortcuts

Space toggles play or pause. F toggles fullscreen. P advances presets. H hides or shows the UI. G opens this guide. R resets the camera.

Compute Shader Implementation

The particle state lives in two pairs of GPU buffers: position/life and velocity/seed. A WebGL2 transform feedback pass runs a custom GLSL vertex shader with rasterization discarded, writes the next state into the alternate buffer pair, then swaps buffers. The render pass consumes the current buffers as point sprites with a custom GLSL material. FFT data is uploaded every frame into a 512-bin R8 texture, while low, mid, high, RMS, centroid, flatness, beat, pointer, and camera features are streamed as uniforms. Scene, trail, bloom, vignette, and grading are separate shader passes over framebuffer textures.
